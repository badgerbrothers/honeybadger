# 执行：按计划实施

## 要执行的计划

读取计划文件：`$ARGUMENTS`

## 执行说明

### 1. 阅读并理解

- 仔细阅读整个计划
- 理解所有任务及其依赖关系
- 记录需要执行的验证命令
- 审查测试策略

### 2. 按顺序执行任务

对“分步任务”中的每一个任务：

#### a. 进入任务上下文

- 确定目标文件和所需操作
- 如果需要修改，先阅读现有相关文件

#### b. 实现任务

- 严格按照详细规格执行
- 保持与现有代码模式一致
- 补充合适的类型标注与文档
- 在适当位置添加结构化日志

#### c. 边做边验证

- 每次修改文件后检查语法
- 确保导入正确
- 确认类型定义完整且正确

### 3. 落实测试策略

在完成实现任务后：

- 创建计划中指定的全部测试文件
- 实现提到的全部测试用例
- 按计划中的测试方案执行
- 确保测试覆盖边界情况

### 4. 运行验证命令

按顺序执行计划中的全部验证命令：

```bash
# 严格按计划中的写法执行每条命令
```

如果任一命令失败：

- 修复问题
- 重新运行命令
- 只有通过后才能继续

### 5. 最终核验

完成前确认：

- [ ] 计划中的所有任务均已完成
- [ ] 所有测试都已创建并通过
- [ ] 所有验证命令均已通过
- [ ] 代码符合项目约定
- [ ] 文档已按需要新增或更新
